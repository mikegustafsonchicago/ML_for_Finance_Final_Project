KORD Linear Regression with Multi-Airport Features: Temperature Prediction Analysis

Model Architecture

This analysis implements multivariate time series regression models to predict temperature at Chicago O'Hare International Airport (KORD) using weather data from multiple airports. The model incorporates both historical data and recent changes in weather patterns across all airports.

Regressors (Predictor Variables)

Our model uses the following variables to predict KORD's temperature:

  • KORD's Own Historical Data:
    • Temperature measurements from the past 24 hours (hourly lags)
    • Temperature measurements from 48 hours ago (2-day lag)
    • Temperature measurements from 120 hours ago (5-day lag)
    • 5-day rolling average temperature (excluding current hour)
  • Other Airports' Current and Historical Data:
    • Temperature measurements and their 24-hour lags
    • Wind speed measurements and their 24-hour lags
    • Wind direction measurements and their 24-hour lags
    • Humidity measurements and their 24-hour lags
    • Dew point measurements and their 24-hour lags
    • Sea level pressure measurements and their 24-hour lags
    • Visibility measurements and their 24-hour lags
    • Cloud coverage measurements and their 24-hour lags
  • 2-Hour Change Features:
    • Temperature changes over the last 2 hours
    • Wind direction changes over the last 2 hours
    • Wind velocity changes over the last 2 hours

Each of these features is calculated for every airport in our dataset, creating a rich set of predictors that capture both local and regional weather patterns.

Data Structure

The dataset contains weather measurements from multiple airports, with each measurement prefixed by its airport code (e.g., KORD, KZZZ). For each airport, we track:

  • Current weather measurements
  • 2-hour changes in key weather variables
  • Historical measurements through lagged features

Note: While we use data from multiple airports, our target variable is always KORD's temperature. The other airports' data serve as potential predictors for KORD's temperature.

Feature Engineering

For each airport in the dataset, we create the following feature groups:

1. Base Weather Features
Feature Name Description Units Time Window
{airport}_temp Temperature measurement at {airport} °C Current
{airport}_windspeed Wind speed measurement at {airport} m/s Current
{airport}_winddirection Wind direction measurement at {airport} degrees Current
{airport}_humidity Humidity measurement at {airport} % Current
{airport}_dew Dew point measurement at {airport} °C Current
{airport}_sealevel Sea level pressure at {airport} hPa Current
{airport}_visibility Visibility measurement at {airport} km Current
{airport}_mincloud Minimum cloud coverage at {airport} ft Current
{airport}_maxcloud Maximum cloud coverage at {airport} ft Current
2. 2-Hour Change Features
Feature Name Description Units Time Window
{airport}_temp_2h_delta Change in temperature over last 2 hours at {airport} °C 2-hour window
{airport}_wind_dir_2h_delta Change in wind direction over last 2 hours at {airport} degrees 2-hour window
{airport}_wind_vel_2h_delta Change in wind velocity over last 2 hours at {airport} m/s 2-hour window
3. Historical Features

For each base feature, we create lagged versions at multiple time intervals:

  • Hourly lags (1-24 hours)
  • 2-day lag (48 hours)
  • 5-day lag (120 hours)

Additionally, for temperature specifically, we include:

  • 5-day rolling average (120-hour window)

Target Variable

The model predicts KORD's temperature at various time horizons:

  • 1 hour ahead
  • 24 hours ahead
  • 5 days ahead
  • 5-day average ahead
  • 30-day average ahead

Training Configuration

  • Train/Test Split: 80/20
  • Random State: 42 (for reproducibility)
  • Validation: Standard train-test split

Model Interpretation

These multivariate regression models capture the complex relationships between weather patterns across multiple airports and temperature at KORD. The models:

  • Use weather conditions from all available airports to predict KORD's temperature
  • Account for recent changes in weather patterns through 2-hour delta features
  • Incorporate historical weather patterns through lagged features
  • Help identify which airports' weather patterns most influence KORD's temperature

Future improvements could include:

  • Feature selection to reduce dimensionality
  • Non-linear models to capture complex relationships
  • Time series specific models (ARIMA, LSTM)
  • Spatial relationship modeling between airports
