KORD Random Forest: Wind Speed Prediction Analysis

Model Architecture

This implementation focuses on predicting wind speeds at Chicago O'Hare International Airport (KORD) using time-lagged weather features to capture temporal patterns in wind behavior.

Model Configuration

  • Target: Next hour's wind speed
  • Time Window: 24-hour historical window for all features
  • Special Features: Wind direction delta calculation to capture directional changes

Feature Details

Feature Name Description Units Time Window
Wind Speed Current and historical wind speed measurements m/s 24-hour lag
Wind Direction Current and historical wind direction measurements degrees 24-hour lag
Wind Direction Delta Change in wind direction between consecutive hours degrees Current
Temperature Current and historical temperature measurements °C 24-hour lag
Humidity Current and historical humidity measurements % 24-hour lag
Dew Point Current and historical dew point measurements °C 24-hour lag
Sea Level Pressure Current and historical pressure measurements hPa 24-hour lag
Visibility Current and historical visibility measurements km 24-hour lag
Cloud Coverage Current and historical min/max cloud coverage ft 24-hour lag

Training Configuration

  • Train/Test Split: 80/20
  • Random State: 42 (for reproducibility)
  • Number of Trees: 100
  • Max Depth: 10
  • Min Samples Split: 5
  • Min Samples Leaf: 2

Model Interpretation

For this specific wind prediction task:

  • The 24-hour lag window captures daily wind patterns at O'Hare
  • Wind direction delta helps identify significant weather pattern changes
  • Feature importance reveals which weather parameters most strongly influence wind speed predictions

Potential improvements specific to KORD:

  • Incorporate runway configuration data
  • Add seasonal feature engineering
  • Include regional weather station data
